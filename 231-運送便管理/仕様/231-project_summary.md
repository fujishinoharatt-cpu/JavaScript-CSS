# 231-運送便管理 カスタマイズ仕様書

## 概要

本ドキュメントは、App 231「運送便管理」に実装されている JavaScript カスタマイズ（`customize.js`）の仕様をまとめたものです。
本アプリは App 227「出荷管理」をデータソースとして連携し、特定の運送便単位での集計や管理を行うことを目的としています。

## 実装ファイル

- `customize.js`: メインロジック
- `bulk_update.js`: 一括更新用汎用スクリプト（設定ファイル `bulk_update_config.js` に依存）
- `customize.css`: スタイル定義（現状は空）

## アプリ連携情報

- **現在のアプリID (App 231)**: 運送便管理
- **連携先アプリID (App 227)**: 出荷管理
- **連携キー**: `出荷日` および `運送便コード`

## 機能一覧 (customize.js)

### 1. 不足分レコード追加

App 227 には存在するが、App 231 にまだ存在しない「運送便」の枠（レコード）を自動作成します。

- **設置場所**: 一覧画面ヘッダ（左側）
- **ボタン名**: `➕ 不足分を追加(冨士)`
- **処理ロジック**:
    1. **App 231 (自アプリ) の全件取得**: 既存の「出荷日」と「運送便コード」の組み合わせをリストアップ。
    2. **App 227 (出荷管理) の検索**: `手配 in ("冨士ファニチア㈱")` の条件でレコードを取得。
    3. **差分抽出**: 227にあって231にない「出荷日 + 運送便コード」の組み合わせを特定。
    4. **レコード追加**: 特定した不足分を App 231 に新規登録する。

### 2. 一括再集計 (冨士)

一覧画面に表示されているレコードに対して、最新の出荷実績を集計して更新します。

- **設置場所**: 一覧画面ヘッダ（右側）
- **ボタン名**: `🔄 一括再集計 (冨士)`
- **更新対象フィールド**:
  - `冨士_個数`
  - `冨士_才数`
- **処理ロジック**:
    1. 一覧上の各レコードの「出荷日」と「運送便コード」を取得。
    2. App 227 を検索し、該当するレコードを取得。
    3. `手配` フィールドに「冨士」が含まれるレコードを集計。
        - **個数**: レコード件数 (Count)
        - **才数**: `才数計`フィールドの合計 (Sum)
    4. 集計結果で App 231 のレコードを一括更新 (PUT) する。

### 3. オリジン集計CSV出力

オリジン分の出荷実績を集計し、CSV形式でダウンロードします。画面上の更新は行いません。

- **設置場所**: 一覧画面ヘッダ（右側）
- **ボタン名**: `📊 ｵﾘｼﾞﾝ集計exp`
- **出力内容**:
  - ヘッダー: `出荷日,運送便コード,オリジン_個数,オリジン_才数`
- **処理ロジック**:
    1. 集計ロジックは「一括再集計」と同様。
    2. ただし、`手配` フィールドに「オリジン」または「ｵﾘｼﾞﾝ」が含まれるレコードを集計対象とする。
    3. 結果をCSVファイルとしてブラウザでダウンロードさせる。

### 4. レコード別集計 (詳細・編集画面)

個別のレコード詳細画面または編集画面で、そのレコード単体の集計値を最新化します。

- **設置場所**: 詳細画面・編集画面ヘッダ
- **ボタン名**: `🔄 レコード別集計(冨士)`
- **処理ロジック**:
    1. 表示中のレコードの「出荷日」と「運送便コード」を使用。
    2. App 227 から最新データを取得・集計（ロジックは一括集計と同じ）。
    3. 画面上のフィールド（`冨士_個数`, `冨士_才数`）に値をセットする。
    4. **注意**: 値をセットするのみで、保存は自動では行われない（ユーザーが保存ボタンを押す必要がある）。

## 補足：集計共通ロジック

`fetchSummaryFrom227` 関数にて App 227 のレコードを集計しています。

- **クエリ条件**: `出荷日 = "YYYY-MM-DD" and 運送便コード = "CODE"`
- **フィルタリング**:
  - **冨士分**: `手配` に "冨士" を含む。
  - **オリジン分**: `手配` に "オリジン" または "ｵﾘｼﾞﾝ" を含む。
- **集計項目**:
  - **個数**: 対象レコードの行数（件数）。`数量`フィールドの合計ではない点に注意。
  - **才数**: `才数計`フィールドの数値合計。小数点以下3桁で四捨五入。
